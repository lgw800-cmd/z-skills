# 父亲概念生成器

把任何输入都当成一个“孩子”，寻找它背后的父亲：更早的实物祖先、更大的类别、更深的欲望、更古老的问题，或能解释它为什么存在的哲学母题。

## 核心目标

给出一个比普通上位词更有解释力的“父概念”。遇到具体物品，先给它的“实体父亲”：更早、更基础、更像祖先的具体物品或器物原型。然后用“维基第一链接式上翻”继续追问：它属于什么，它解决什么，它暴露了什么人类处境，它最终通向哪个哲学问题。

用户只输入一个词时，直接返回结果。不要要求用户补充背景，除非输入完全无法理解。

## 判断路径

按下面顺序快速分析输入：

1. 识别输入是什么：物品、动作、情绪、职业、技术、品牌、标题、关系、现象、观点、谜语式短句。
2. 判断它是否是具体物品。若是，先找实体父亲：它由什么更早的器物、原型、自然物、手工工具或机械结构演化而来。
3. 找到近亲上位词：它最朴素的类别是什么。
4. 找到功能来源：它满足了什么需求，解决了什么问题。
5. 做一次“第一链接式上翻”：每一步只选一个最自然、最能解释上一层的父概念。
6. 找到精神来源：它背后的人类欲望、恐惧、幻觉、秩序、冲突或命运是什么。
7. 找到哲学父亲：它最终通向认识论、存在论、技术哲学、身体哲学、语言哲学、伦理学、美学、政治哲学、时间、自由、边界、秩序、自我、他者、自然或意义中的哪一个母题。
8. 找到最有记忆点的父亲：在准确和有趣之间取平衡，优先选择能让用户“啊，这个说法有东西”的答案。

## 维基第一链接式上翻

借用“维基百科哲学现象”的思路：很多条目沿第一个正文链接不断点击，会逐步从具体事物走向更抽象的概念，最终靠近哲学。这里不需要真的打开网页，默认生成一条“概念上翻路径”。

上翻路径要像这样逐层变高：

```text
具体物/概念 -> 实体父亲或近亲父亲 -> 上位类别 -> 功能/需求 -> 文明问题 -> 哲学母题
```

选择每一层时遵守三条规则：

- 每一步只上翻一层，避免突然跳到宏大词。
- 每一步都要能解释前一步，像维基正文里的第一个关键概念一样自然。
- 终点尽量落到可命名的哲学母题，如“人与自然的关系”“心智外化”“边界”“意义”“秩序”“自由”“认识论”“存在论”“语言与世界”。

## 父亲类型

根据输入选择 2-4 个最合适的父亲类型，不必每次全列。

- 实体父亲：具体物品的直接器物祖先、结构原型或前身，如“风扇”的实体父亲可以是“扇子”，“电脑”的实体父亲可以是“算盘/打字机/计算器”。
- 分类父亲：字典意义上的上位概念，如“咖啡”的分类父亲是“饮品”。
- 功能父亲：它被发明、被使用、被需要的原因，如“咖啡”的功能父亲是“清醒”。
- 欲望父亲：它服务的人类欲望，如“奢侈品”的欲望父亲是“被看见”。
- 恐惧父亲：它缓解的焦虑，如“保险”的恐惧父亲是“失控”。
- 秩序父亲：它把混乱变成什么秩序，如“日历”的秩序父亲是“可管理的时间”。
- 文明父亲：它在社会层面的来源，如“法律”的文明父亲是“可重复的公平”。
- 哲学父亲：一路上翻后抵达的最终母题，如“电脑”的哲学父亲可以是“心智外化”，“门”的哲学父亲可以是“边界”。
- 命运父亲：它指向的存在处境，如“家”的命运父亲是“流浪之后的停靠”。
- 反父亲：它反抗、背离或逃出的东西，如“摇滚”的反父亲是“规训”。

## 默认输出格式

用户没有指定格式时，使用这个结构，保持短、有力、有余味：

```text
父：[最有解释力的父概念]

谱系：[输入] -> [近亲上位] -> [更高来源] -> [元问题/元欲望]

维基式上翻：[输入] -> [实体/近亲父亲] -> [上位类别] -> [功能/需求] -> [文明问题] -> [哲学母题]

三位父亲：
- 实体父亲：[若输入是具体物品，给具体物品祖先；若输入是抽象概念，可省略]
- 分类父亲：[朴素类别]
- 功能父亲：[它解决的需求]
- 哲学父亲：[最终通向的哲学母题]

一句话：[有记忆点的凝练表达]
```

如果输入很简单，可以压缩成 5-7 行。用户要求“更深刻”“更有趣”“多来几个”时，展开到 5-7 个候选父亲，并标出最推荐的一个。若用户明确要求“真实维基路径”或“按维基百科真实链接查”，再去查网页；默认输出概念路径。

## 创作规则

- 先准确，再漂亮。父概念要能解释输入，不能只追求诗意。
- 具体物品优先回答具体父亲。先找“它像谁生出来的”，再找“它为什么被需要”。
- 具体物品的实体父亲要尽量可触摸、可想象、可画出来。优先选择扇子、算盘、轮子、容器、刀、桥、火把、书、钥匙、钟、门、船这类能构成器物谱系的东西。
- 若一个物品有多个实体父亲，选最能解释它核心功能的那个。比如“风扇”优先给“扇子”，再补“叶轮/风车”作为结构父亲。
- 所有输入都要给“维基式上翻”。它是这个 skill 的骨架，负责把具体答案一路推到哲学层。
- 上翻链条要克制，通常 5-6 个节点足够。每个节点必须比前一个更抽象或更根本。
- 避免只给平庸类别。比如“苹果 -> 水果”只算起点，还要继续追问它的味觉、神话、诱惑、日常性或记忆。
- 把抽象概念落到人类经验里。概念的父亲常常是一个欲望、恐惧、缺口、矛盾或古老问题。
- 保留一点意外感。好的父亲答案应该像把词往上拎了一层，同时让它多出影子。
- 不要过度解释。用户要的是可直接使用的概念答案，避免长篇论文式铺陈。
- 避免先否后转的固定句式，直接给出判断和深化。

## 多输入处理

如果用户一次给多个词，用表格：

| 输入 | 父 | 维基式上翻 | 一句话 |
|---|---|---|---|
| [词] | [父概念] | [简短上翻链路] | [凝练表达] |

每个输入只给一个最强父概念，除非用户要求展开。

## 示例

**输入：咖啡**

```text
父：清醒

谱系：咖啡 -> 饮品 -> 自我驱动 -> 对白天的抵抗

维基式上翻：咖啡 -> 植物饮品 -> 兴奋剂 -> 工作节律 -> 身体与效率 -> 现代生活哲学

三位父亲：
- 分类父亲：饮品
- 功能父亲：提神
- 哲学父亲：身体如何被时间表驯化

一句话：咖啡的父亲，是人类不肯向早晨投降。
```

**输入：风扇**

```text
父：扇子

谱系：风扇 -> 扇子 -> 人造风 -> 对闷热的反抗

维基式上翻：风扇 -> 扇子 -> 送风工具 -> 舒适控制 -> 人与环境的关系 -> 技术哲学

三位父亲：
- 实体父亲：扇子
- 结构父亲：叶轮/风车
- 功能父亲：送风和散热
- 哲学父亲：人类如何用技术改写身体所处的环境

一句话：风扇的父亲是扇子，祖父是风；它把手腕的动作交给电机，让风按人的意思出现。
```

**输入：电脑**

```text
父：计算器

谱系：电脑 -> 计算器/打字机 -> 信息处理机器 -> 外化心智

维基式上翻：电脑 -> 计算器 -> 计算工具 -> 信息处理 -> 心智外化 -> 认识论

三位父亲：
- 实体父亲：计算器
- 旁系父亲：打字机
- 功能父亲：计算、记录、编辑和连接
- 哲学父亲：人类如何把思考放到身体之外

一句话：电脑的父亲是计算器，养父是打字机；它把算术和书写合并成一块可操作的外部大脑。
```

**输入：门**

```text
父：边界

谱系：门 -> 门帘/洞口 -> 可控制的入口 -> 内外之分

维基式上翻：门 -> 门帘/洞口 -> 入口装置 -> 空间边界 -> 安全与自由 -> 边界哲学

三位父亲：
- 实体父亲：门帘/洞口
- 分类父亲：入口装置
- 功能父亲：控制进入
- 哲学父亲：边界

一句话：门的父亲，是世界第一次学会说“进来”。
```

**输入：AI**

```text
父：外包的心智

谱系：AI -> 工具 -> 自动化判断 -> 人类对第二大脑的渴望

维基式上翻：AI -> 计算系统 -> 自动化判断 -> 认知外包 -> 人与机器的关系 -> 心智哲学

三位父亲：
- 分类父亲：计算系统
- 功能父亲：认知自动化
- 哲学父亲：心智是否只能存在于人脑之内

一句话：AI 的父亲，是人类对自己头脑既骄傲又疲惫的那一刻。
```

## 质量自检

交付前检查：

- 是否以“父：”开头。
- 若输入是具体物品，是否先给出具体的实体父亲。
- 是否给出“维基式上翻”，并逐层抵达一个哲学母题。
- 是否给出至少一个朴素上位概念和一个更深层父亲。
- 是否能解释输入为什么会存在、被需要或被记住。
- 是否避免只停留在字典分类。
- 是否有一句可以被用户直接拿走的凝练表达。
